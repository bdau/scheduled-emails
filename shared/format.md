# Shared email format (house look)

Presentation layer for the email framework. Fetched at run time by every scheduled email. Any email that wants the house look assembles its HTML by:

1. Rendering its own section blocks (the email's brief owns these).
2. Concatenating them into {{BODY_CONTENT}} of the shell below.
3. Substituting {{EMAIL_TITLE}}, {{BRAND}}, {{DATE}}, and {{FOOTER_TEXT}}.

Substitute only the {{...}} placeholders. Never edit an inline style attribute, colour, font, padding, or width. Email clients are unforgiving, so the styles are inline by necessity.

<design_tokens>
Reference only. The values are already baked into the templates below; this list is so a new email stays on-palette when it builds its own components.

Backgrounds:  page #080b11 | card #0f141c | positive tint #0c1a15 | negative tint #1a0f12 | neutral tint #191509
Borders/rules: #1e2733
Text:         primary #d5dce6 | muted #7c8794
Accents:     positive #16c784 | negative #ea3943 | neutral #e0a417 | source-link #155DFC | in-body link #4aa8ff
Type:        headings/labels/mono -> 'SFMono-Regular',Consolas,'Liberation Mono',Menlo,monospace
             body -> -apple-system,'Segoe UI',Roboto,Helvetica,Arial,sans-serif
Container:   640px, border-radius 3px, 28px side padding
</design_tokens>

<shell>
{{EMAIL_TITLE}} appears twice (page title and masthead) - use the same string in both. {{BRAND}} comes from the email's config. {{DATE}} formatted like "9 July 2026". {{BODY_CONTENT}} is the concatenation of the email's own section &lt;tr&gt; blocks. {{FOOTER_TEXT}} is a one-line disclaimer supplied by the email's config (a generic default is given below).
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width,initial-scale=1">
  <meta name="color-scheme" content="dark">
  <meta name="supported-color-schemes" content="dark">
  <title>{{EMAIL_TITLE}}</title>
</head>
<body style="margin:0;padding:0;background:#080b11;">
<table role="presentation" width="100%" cellpadding="0" cellspacing="0" border="0" bgcolor="#080b11" style="background:#080b11;">
<tr><td align="center" style="padding:24px 12px;">
<table role="presentation" width="640" cellpadding="0" cellspacing="0" border="0" bgcolor="#0f141c" style="width:640px;max-width:640px;background:#0f141c;border-radius:3px;border:1px solid #1e2733;">

  <!-- MASTHEAD -->
  <tr><td style="padding:28px 28px 8px 28px;">
    <div style="font-family:'SFMono-Regular',Consolas,'Liberation Mono',Menlo,monospace;font-size:17px;font-weight:700;letter-spacing:2.5px;text-transform:uppercase;color:#7c8794;">{{BRAND}}</div>
    <div style="font-family:'SFMono-Regular',Consolas,'Liberation Mono',Menlo,monospace;font-size:36px;font-weight:700;color:#d5dce6;margin:6px 0 2px 0;">{{EMAIL_TITLE}}</div>
    <div style="font-family:-apple-system,'Segoe UI',Roboto,Helvetica,Arial,sans-serif;font-size:20px;color:#7c8794;">{{DATE}}</div>
    <div style="height:2px;background:#1e2733;margin:18px 0 0 0;"></div>
  </td></tr>

{{BODY_CONTENT}}

  <!-- FOOTER -->
  <tr><td style="padding:20px 28px 30px 28px;">
    <div style="height:1px;background:#1e2733;margin:0 0 14px 0;"></div>
    <div style="font-family:-apple-system,'Segoe UI',Roboto,Helvetica,Arial,sans-serif;font-size:17px;line-height:1.6;color:#7c8794;">{{FOOTER_TEXT}}</div>
  </td></tr>

</table>
</td></tr></table>
</body>
</html>
```
Default {{FOOTER_TEXT}}: "Generated by your personal assistant agent. Not investment advice."
</shell>

<section_wrapper>
Each section the email renders is one table row. Put the section_header first, then the section's content.
```html
<tr><td style="padding:24px 28px 4px 28px;">
  {{SECTION_HEADER}}
  {{SECTION_CONTENT}}
</td></tr>
```
</section_wrapper>

<section_header>
Uppercase mono heading with an underline rule. {{TITLE}} is the section name.
```html
<div style="font-family:'SFMono-Regular',Consolas,'Liberation Mono',Menlo,monospace;font-size:20px;font-weight:700;letter-spacing:2.5px;text-transform:uppercase;color:#d5dce6;padding:0 0 10px 0;border-bottom:2px solid #1e2733;margin:0 0 18px 0;">{{TITLE}}</div>
```
Optional status dot: to prefix the title with a coloured square (e.g. a Positive/Negative section), insert this span immediately after the opening &lt;div ...&gt; and before {{TITLE}}. Swap the background colour for the accent you want (positive #16c784, negative #ea3943, neutral #e0a417).
```html
<span style="display:inline-block;width:8px;height:8px;border-radius:2px;background:#16c784;margin-right:8px;vertical-align:middle;"></span>
```
</section_header>

<source_link>
Use for EVERY source citation link, in every email and every section. {{NAME}} is the site name only (e.g. WSJ, FT, Bloomberg, Reuters). display:inline-block is required so padding renders in Outlook. Separate consecutive source links with a single space.
```html
<a href="{{URL}}" style="display:inline-block;background:#155DFC;color:#ffffff;padding:2px;border-radius:3px;text-decoration:none;">{{NAME}}</a>
```
</source_link>

<link>
Use ONLY for an in-body prose hyperlink (not a source citation). The inline colour keeps it legible on the dark background.
```html
<a href="{{URL}}" style="color:#4aa8ff;text-decoration:underline;">{{NAME}}</a>
```
</link>

<notes>
- Reusable-vs-bespoke rule: keep a component here only once a second email actually uses it. Components used by a single email live in that email's brief. Promote to this file when the second consumer appears, not before.
- {{DATE}} in bash: `date +"%-d %B %Y"`.
</notes>
